## How to Contribute

### Adding a New Resource

1. **Fork** the repository
2. **Add your resource** to the appropriate section in `README.md`
3. **Follow the format**: `- [Name](link) - Short description.`
4. **Submit a Pull Request** with a clear title and description

### Format Requirements

Each entry should follow this format:

```markdown
- [Tool Name](https://github.com/org/repo) - Brief description ending with a period.
```

**Guidelines:**
- Keep descriptions concise (one sentence, under 100 characters)
- Start descriptions with a capital letter
- End descriptions with a period
- Use proper spelling and grammar
- Maintain alphabetical order within sections

### Quality Standards

Before submitting, please ensure your resource:

- [ ] Has a clear README with installation instructions
- [ ] Is related to OHDSI/OMOP CDM
- [ ] Has been updated within the last 2 years (exceptions for stable tools)
- [ ] Has a license
- [ ] Actually works (you've tested it or can vouch for it)

### Suggesting Changes

If you want to suggest a structural change, new category, or have questions:

1. Open an [Issue](../../issues) first
2. Describe your suggestion
3. Wait for discussion before making a PR

### Code of Conduct

- Be respectful and inclusive
- Focus on the content, not the contributor
- Assume good intentions

## Questions?

Feel free to open an issue if you have any questions about contributing.
